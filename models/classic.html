We provide the implementation of some classical and simple models. Altough the implementation is usually straigh-forward, we provide it to be used to quick experiments and to provide a simple interface.

Implementation of the classic LogisticRegression model. If you want to use a model with hidden layers, consider using the FeedForward.

  • dropout (float) â if defined, apply a dropout in the incoming data.

class cogitare.models.classic.FeedForward(input_size, num_layers, hidden_size, activation=None, in_dropout=0.0, hidden_dropout=0.0, loss_function=None, bias=True, use_cuda=None)[source]Â¶

Implementation of the classic FeedForward (a.k.a. MLP - Multi-Layer Perceptron). This is a sequence of Linear layers, with dropout and activation functions in between.

Parameters:
  • input_size (int) â size of the input data. The input data must have the shape \(NxD\), where N is the batch size, and D is the input_size.
  • num_layers (int) â number of hidden layers in the model. For each layer, you can define a dropout value to apply in its input, and its activation function.
  • hidden_size (int, tuple, list) â if int, the dimension of the hidden layer for all layers. If a list or a tuple, you can define de hidden layer size for each hidden layer.
  • activation (torch.nn.Module) â a PyTorch activation function. By default, it uses tanh in all layers but the last one, which uses a log-softmax.
  • in_dropout (float) â if defined, the dropout in the input data.
  • hidden_dropout (float, tuple, list) â dropout to the input of the hidden layer. If is a float, all hidden layers will have the same dropout applyed to the incoming data. If a list or a tuple, you can define custom dropout values per layer.
  • loss_function (torch.nn.Module) â feed-forward loss function. By default, it uses the negative log-likelihood.
  • bias (bool, tuple, list) â if True (the default), adds the bias term in the linear layer. If tuple or a list, the bias term can be specified per layer.
  • use_cuda (bool) â if True, move the model parameters and data to cuda.
